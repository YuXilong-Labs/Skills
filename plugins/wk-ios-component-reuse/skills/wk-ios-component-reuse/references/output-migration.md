# Migration 输出模板与迁移 Playbook

## 迁移映射表模板

| 重复实现点 | 旧调用/API | 目标组件 | 目标 API | 关键差异 | 风险 | 批次 | 验证项 |
|---|---|---|---|---|---|---|---|

**填写要求：**
- `关键差异` 至少覆盖：参数/返回值/线程/错误处理（按适用项填写）
- `风险` 使用：高 / 中 / 低
- `批次` 必须可执行，不要写"全量一次性替换"
- `验证项` 至少包含 1 个功能验证 + 1 个错误路径验证（关键场景）

## 迁移 Playbook

### 1. 盘点阶段

- 列出重复实现点（文件/能力类型/调用方）
- 标记影响范围（模块数、调用次数、是否核心流程）
- 标记风险（用户路径、支付/登录/上传等关键链路）

### 2. 映射阶段

为每个重复点建立映射：
- 旧实现能力 → 目标组件/API
- 参数差异
- 返回值差异
- 线程/回调语义差异
- 错误处理差异
- 埋点/日志/监控差异

### 3. 分批替换阶段（推荐顺序）

1. **批次 1：** 低风险内部场景
2. **批次 2：** 非核心用户路径
3. **批次 3：** 核心链路（需重点验证）

**策略：**
- 优先新增兼容适配层（必要时）
- 逐场景替换并验证
- 每批次结束后评估是否推进下一批

### 4. 验证阶段

**必做：**
- 功能回归
- 错误路径验证（超时、失败、空响应）
- 日志/埋点/监控一致性检查

**建议：**
- 性能基线对比（启动、内存、耗时）
- 崩溃告警观察窗口

### 5. 回滚策略

**回滚触发条件示例：**
- 核心流程回归失败
- 错误率明显上升
- 崩溃率异常波动
- 关键埋点丢失

**回滚动作：**
- 回退当前批次调用点
- 保留适配层但关闭新路径开关（如有）
- 记录问题与阻塞条件

### 6. 收口阶段

- 删除废弃封装/调用入口
- 更新项目规范与示例
- 记录未完成项与后续计划
