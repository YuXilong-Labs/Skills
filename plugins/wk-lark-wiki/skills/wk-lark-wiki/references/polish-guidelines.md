# AI 润色规则

## 文档结构模板

润色后的组件文档必须遵循以下章节顺序：

```
# 组件名

> 一句话摘要（callout 格式）

详细描述（1-3 段）

**API 数量：** N | **源文件数：** M

---

以 tree 格式展示组件源码目录结构，每个文件/目录附注释说明用途。
```

## 描述生成规则

遇到 `*暂无说明*` 或空描述时，按以下优先级生成描述：

### 0. 从源码实现验证（最高优先级）

对关键 API（初始化方法、核心业务方法、状态管理属性），必须读取源码实现验证描述准确性：

| 场景 | 处理方式 |
|------|----------|
| 注释说"同步"但实现为异步 | 修正为异步，标注实际调用方式 |
| 参数有合法值范围（如 0.0-1.0） | 补充到描述中 |
| 方法有线程安全要求 | 在描述中标注线程安全约束 |
| 调用有前置条件（如必须先 setup） | 补充前置条件说明 |
| 方法有副作用（如修改全局状态） | 标注副作用 |

**优先级：** 源码实现 > 注释声明 > 方法名推断

> 不确定的描述标注 `*需确认 — 根据源码推断*`，不臆测功能。

### 1. 从方法名推断

ObjC 的 selector 是自文档化的，直接翻译为中文：

| 声明 | 生成描述 |
|------|----------|
| `- (void)startRecording` | 开始录制 |
| `+ (instancetype)sharedInstance` | 获取共享实例（单例） |
| `- (BOOL)isPlaying` | 返回当前是否正在播放 |
| `- (void)setVolume:(float)volume` | 设置音量 |

### 2. 从参数和返回类型推断

```objc
// 参数名 + 类型 → 功能描述
- (void)sendMessage:(NSString *)content toUser:(NSString *)userId;
// → 向指定用户发送消息
```

### 3. 从属性声明推断

```objc
@property(nonatomic, weak) id<XXDelegate> delegate;
// → 代理对象，用于接收回调通知

@property(nonatomic, assign, readonly) BOOL isConnected;
// → 当前是否已连接（只读）
```

### 4. 描述质量要求

- 1-2 句话，不超过 50 字
- 说清"做什么"，不解释"怎么做"
- 避免重复方法名本身（❌ "此方法用于发送消息" → ✅ "向指定用户发送文本消息"）
- 如果确实无法从声明推断，写 `功能待补充（需查看源码实现）`

## 飞书 Wiki 格式最佳实践

### 表格

- 列数 ≤ 4 列（飞书移动端超过 4 列阅读体验差）
- 表格内避免代码块（飞书渲染异常），改用 `` `inline code` ``
- 保持每行列数一致（列数不对齐会导致渲染错误）
- 表格前后各空一行

### 标题层级

- `# H1` — 仅组件名使用
- `## H2` — 类/协议名
- `### H3` — 属性/方法/Category 分组
- `#### H4` — 单个 API 条目

> 不要使用 H5 及以下，飞书渲染区分度低。

### 代码块

- 始终指定语言标签：`` ```objc ``、`` ```swift ``
- 代码块内不留前后空行（飞书会渲染为额外空行）
- 单行声明：用 inline code 即可，不必使用代码块
- 代码块长度 ≤ 15 行（过长时截取关键部分）

### 引用块

- 组件摘要用引用块：`> 摘要内容`
- 废弃警告用引用块：`> ⚠️ **已废弃** — 说明`
- AI 生成的描述用引用块标注：`> *AI 生成* — 说明`
- 不要嵌套引用块（飞书不支持）

### 列表

- 最多 2 层嵌套（飞书深层嵌套缩进异常）
- 参数列表 ≥ 3 个时用表格，< 3 个用无序列表
- 避免在列表项内放代码块

### 分隔线

- 类/协议章节之间用 `---` 分隔
- 不要在标题紧下方放分隔线

## 质量检查清单

润色完成后，逐项确认：

- [ ] 文件目录架构图存在且每个文件有注释
- [ ] 无 `*暂无说明*` 残留（全部替换为有意义的描述）
- [ ] 关键 API 描述已通过源码验证
- [ ] 所有表格列数一致（无错位行）
- [ ] 所有代码块有语言标签（`objc` 或 `swift`）
- [ ] 无孤立标题（标题下必须有内容）
- [ ] 标题层级正确（H1 仅组件名，H2 仅类/协议，H3/H4 成员）
- [ ] emoji 标签统一：📦 类 / 📋 协议 / 🔧 方法 / 📌 属性 / 📎 枚举
- [ ] 废弃标注使用引用块格式
- [ ] "快速开始"章节存在且包含实用示例
- [ ] API 概览表格在详细章节之前
- [ ] 文件末尾有生成时间戳

## 润色幂等性规则

多次润色必须收敛到稳定结果：

1. 按模板结构输出，不要"在现有基础上追加"
2. 描述生成遵循固定规则，不要每次换说法
3. 格式化规则是确定性的（表格/列表/代码块的选择有明确阈值）
4. 如果内容已经符合规则，保持原样不改动
